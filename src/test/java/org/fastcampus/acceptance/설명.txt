
인수테스트에서는 기존과 다르게 DB를 사용해서 만듬.

이럴때 가장 유의해야 할 점은?
템플릿에서 어떤 작업을 해야할까??

--> 테스트마다 독립성이 보장되어야 함!!!!!!!!!

테스트 유지보수 측면에서 가장 중요한 것은 환경에 따라서 혹은 어떤 테스트를 함께 진행한다고 해서 다른 결과가 나오면 안된다는 것임.

이를 위해 템플릿을 제작하는데 해당 템플릿은 이전 디비 테스트 환경을 싹 지워주는 클래스를 먼저 만들어주려고 함.

클래스를 만들기 위한 두가지 작업 => DB 테이블을 정리하는 클래스를 만들겠음. (DatabaseCleanUp)
1. 엔티티로 만든 테이블 이름을 가져와야 함.
2. 그것들을 전부 삭제하는 쿼리문 발송.
